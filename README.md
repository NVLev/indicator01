# 🩻 ИИ-сервис анализа КТ исследований грудной клетки

## 📋 Краткое описание
Сервис для автоматического анализа компьютерной томографии (КТ) грудной клетки с использованием искусственного интеллекта. Система принимает DICOM-исследования в формате ZIP, обрабатывает их с помощью ML-модели и предоставляет детализированные отчеты с визуализацией областей патологии.

---

## 🎯 Основные возможности

### ✅ Реализовано
- Загрузка исследований — прием ZIP-архивов с DICOM файлами  
- Автоматическая обработка — асинхронная обработка через Celery workers  
- AI-анализ — определение вероятности патологий с помощью ML-модели
- Excel отчеты — генерация структурированных отчетов по ТЗ  
- REST API — полный набор эндпоинтов для интеграции  
- JWT аутентификация — безопасный доступ к API  
- Демо-режим — готовый интерфейс для тестирования без фронтенда  

## Дополнительно

### ✅ Массовая загрузка

- Отдельная форма для массовой загрузки с множественным выбором файлов
- Валидация - ограничение до 20 файлов с предупреждением
- Визуальный прогресс - индикатор выполнения и счетчик файлов
- Подробная статистика - показ успешных и неудачных загрузок
- ID исследований - отображение созданных ID для удобства отслеживания

### Heatmap визуализация
### Защита от "галлюцинаций" модели - RAG система VerificationEngine

### ⚠️ Ограничения
- Поддерживается только формат ZIP с DICOM файлами  
- Время обработки: до 10 минут  
- Требуется GPU для оптимальной работы ML-модели

## 🚀 Быстрый старт

1. **Скачивание архива с проектом**  
   Получите архив проекта (например, с Google Disk) и распакуйте его:  
   ```bash
   unzip indicator01.zip
   cd indicator01
   cp .env.template .env
   ```

2. **Запуск системы**
   ```bash
   # Полный запуск с миграциями БД
   docker compose --profile migrations up -d

   # Или пошагово:
   docker compose up pg redis -d
   docker compose --profile migrations run --rm alembic
   docker compose up -d
   ```

3. **Проверка работоспособности**
   ```bash
   # Проверка статуса сервисов
   docker compose ps

   # Просмотр логов
   docker compose logs backend
   ```

4. **Демо-режим (быстрое тестирование)**
   - Откройте http://localhost:8000/demo  
   - Используйте готовый демо-интерфейс  
   - Загрузите тестовый ZIP с DICOM файлами  
   - Просматривайте результаты и отчеты  

5. **Пример использования через API**
   ```bash
   # Получение токена
   curl -X POST http://localhost:8000/auth/login \
     -H "Content-Type: application/json" \
     -d '{"email":"demo@test.com","password":"demo_password_123"}'

   # Загрузка исследования
   curl -X POST http://localhost:8000/studies/upload \
     -H "Authorization: Bearer <token>" \
     -F "file=@research.zip"
   ```

## 👤 Руководство пользователя

### Демо-режим
- Доступ: http://localhost:8000/demo  
- Авторизация: автоматическая (демо-пользователь создается при запуске)

Просмотр результатов:
- Список всех исследований: http://localhost:8000/demo/results  
- Детали исследования: нажмите "Подробнее"  
- Excel отчет: кнопка "📊 Excel отчет"  
- Heatmap: кнопка "🖼️ Heatmap"
